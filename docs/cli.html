Command Line Interface

seismic demult

Split multiplexed FASTQ files by their barcodes.

seismic demult [OPTIONS] FASTA

Options

-x, --fastqp <fastqp>

FASTQ files of paired-end reads separated into 2 files

--phred-enc <phred_enc>

Phred score encoding in FASTQ and SAM/BAM/CRAM files

-s, --sections-file <sections_file>

CSV file of sections by name, reference, and coordinates/primers

--barcode-start <barcode_start>

index of start of barcode

--barcode-length <barcode_length>

length of barcode

-o, --out-dir <out_dir>

Where to output all finished files

--parallel-demultiplexing <parallel_demultiplexing>

Whether to run demultiplexing at maximum speed by submitting multithreaded grep functions

--clipped <clipped>

Designates the amount of clipped patterns to search for in the sample, will raise compution time

--mismatch-tolerence <mismatch_tolerence>

Designates the allowable amount of mismatches allowed in a string and still be considered a valid pattern find. will increase non-parallel computation at a factorial rate. use caution going above 2 mismatches. does not apply to clipped sequences.

--index-tolerance <index_tolerance>

Designates the allowable amount of distance you allow the pattern to be found in a read from the reference index

--demulti-overwrite <demulti_overwrite>

desiginates whether to overwrite the grepped fastq. should only be used if changing setting on the same sample

Arguments

FASTA

Required argument

seismic align

Trim FASTQ files with Cutadapt (optional), validate them with FASTQC (optional), and align them to a FASTA file with Bowtie2.

seismic align [OPTIONS] FASTA

Options

-x, --fastqp <fastqp>

FASTQ files of paired-end reads separated into 2 files

-y, --fastqi <fastqi>

FASTQ files of paired-end reads interleaved in 1 file

-z, --fastqs <fastqs>

FASTQ files of single-end reads

-X, --dmfastqp <dmfastqp>

Demultiplexed FASTQ files of mate 1 and mate 2 reads

-Y, --dmfastqi <dmfastqi>

Demultiplexed FASTQ files of paired-end reads interleaved in one file

-Z, --dmfastqs <dmfastqs>

Demultiplexed FASTQ files of single-end reads

--phred-enc <phred_enc>

Phred score encoding in FASTQ and SAM/BAM/CRAM files

-o, --out-dir <out_dir>

Where to output all finished files

-t, --temp-dir <temp_dir>

Where to write all temporary files

--force, --no-force

Force all tasks to run, even those whose output files already exist

--save-temp, --no-save-temp

Whether to save temporary files when the program exits

--parallel, --serial

Whether to process input files in parallel or in series

--max-procs <max_procs>

Maximum number of simultaneous processes

--fastqc, --no-fastqc

Whether to check quality of FASTQ files

--qc-extract, --qc-no-extract

Whether to unzip FASTQC reports

--cut, --no-cut

Whether to trim reads with Cutadapt before alignment

--cut-a1 <cut_a1>

3’ adapter for read 1

--cut-g1 <cut_g1>

5’ adapter for read 1

--cut-a2 <cut_a2>

3’ adapter for read 2

--cut-g2 <cut_g2>

5’ adapter for read 2

--cut-O <cut_o>

Minimum overlap of read and adapter

--cut-e <cut_e>

Error tolerance for adapters

--cut-q1 <cut_q1>

Phred score for read 1 quality trimming

--cut-q2 <cut_q2>

Phred score for read 2 quality trimming

--cut-m <cut_m>

Discard reads shorter than this length after trimming

--cut-indels, --cut-no-indels

Whether to allow indels in adapters

--cut-discard-trimmed, --cut-keep-trimmed

Whether to discard reads in which an adapter was found

--cut-discard-untrimmed, --cut-keep-untrimmed

Whether to discard reads in which no adapter was found

--cut-nextseq, --cut-no-nextseq

Whether to trim high-quality Gs from 3’ end

--bt2-local, --bt2-end-to-end

Whether to perform local or end-to-end alignment. Use local with fragmented samples, on which end-to-end yields false positives at read ends. Use end-to-end with amplicon-based samples, on which local yields false negatives at read ends.

--bt2-discordant, --bt2-no-discordant

Whether to output discordant alignments

--bt2-mixed, --bt2-no-mixed

Whether to align individual mates of unaligned pairs

--bt2-dovetail, --bt2-no-dovetail

Whether to treat dovetailed mate pairs as concordant

--bt2-contain, --bt2-no-contain

Whether to treat nested mate pairs as concordant

--bt2-unal, --bt2-no-unal

Whether to output unaligned reads

--bt2-I <bt2_i>

Minimum fragment length for valid paired-end alignments

--bt2-X <bt2_x>

Maximum fragment length for valid paired-end alignments

--bt2-score-min-e2e <bt2_score_min_e2e>

Minimum score for a valid end-to-end alignment

--bt2-score-min-loc <bt2_score_min_loc>

Minimum score for a valid local alignment

--bt2-i <bt2_s>

Seed interval

--bt2-L <bt2_l>

Seed length

--bt2-gbar <bt2_gbar>

Minimum distance of a gap from end of a read

--bt2-D <bt2_d>

Maximum number of failed seed extensions

--bt2-R <bt2_r>

Re-seed up to this many times

--bt2-dpad <bt2_dpad>

Width of padding on alignment matrix, to allow gaps

--bt2-orient <bt2_orient>

Valid orientations of paired-end mates

Options

fr | rf | ff

--min-mapq <min_mapq>

Ignore alignments whose mapping quality is less than this number

-n, --min-reads <min_reads>

Do not analyze datasets with fewer than this minimum number of reads

--cram, --bam

Output alignment maps in BAM or CRAM format. BAM format has faster read/write speeds, while CRAM format yields smaller files.

seismic mask

Select a section of the reference, define which relationships count as mutations, and filter out unusable positions and reads.

seismic mask [OPTIONS] [INPUT_PATH]...

Options

-c, --coords <coords>

Reference name, 5’ end, and 3’ end of a section; coordinates are 1-indexed and include both ends

-p, --primers <primers>

Reference name, forward primer, and reverse primer of a section; reverse primer must be given 5’ to 3’

--primer-gap <primer_gap>

Number of bases to leave as a gap between the end of a primer and the end of the section

-s, --sections-file <sections_file>

CSV file of sections by name, reference, and coordinates/primers

--count-del, --discount-del

Whether to count deletions as mutations

--count-ins, --discount-ins

Whether to count insertions as mutations

--discount-mut <discount_mut>

Do not count a specific type of mutation

--exclude-polya <exclude_polya>

Exclude stretches of consecutive A bases of at least this length. If 0, exclude none.

--exclude-gu, --include-gu

Exclude positions with G and U bases (which DMS methylates very weakly at physiological pH).

--exclude-pos <exclude_pos>

Exclude arbitrary positions, given as (reference, position).

--min-finfo-read <min_finfo_read>

Filter reads with less than this fraction of informative positions.

--max-fmut-read <max_fmut_read>

Filter reads with more than this fraction of mutated positions.

--min-mut-gap <min_mut_gap>

Filter out reads with any pair of mutations that are separated by fewer than this number of non-mutated bases. If 0, filter out no reads.

--min-ninfo-pos <min_ninfo_pos>

Filter positions with less than this number of informative reads.

--max-fmut-pos <max_fmut_pos>

Filter positions with more than this fraction of mutated reads.

--brotli-level <brotli_level>

Compression level for brotli: 0 (fastest, but worst compression) to 11 (best compression, but slowest)

--max-procs <max_procs>

Maximum number of simultaneous processes

--parallel, --serial

Whether to process input files in parallel or in series

--force, --no-force

Force all tasks to run, even those whose output files already exist

Arguments

INPUT_PATH

Optional argument(s)

seismic cluster

Cluster reads from ‘mask’ using Expectation-Maximization to find alternative structures in the RNA ensemble.

seismic cluster [OPTIONS] [INPUT_PATH]...

Options

-k, --max-clusters <max_clusters>

End the clustering step after attempting this number of clusters, even if it yields the best (smallest) BIC observed thus far. If 0, do not run clustering.

--min-nmut-read <min_nmut_read>

Ignore reads with less than this number of mutated positionsduring clustering.

-e, --em-runs <em_runs>

Run clustering this many times for each number of clusters, randomly initializing each run.

--em-thresh <em_thresh>

Consider an EM run to have converged when the log likelihood value has increased by less than this threshold between two consecutive iterations.

--min-em-iter <min_em_iter>

Run every EM run for at least this number times k iterations, even if the likelihood value has already converged.

--max-em-iter <max_em_iter>

Stop every EM run after this number times k iterations, even if the likelihood value has not yet converged.

--brotli-level <brotli_level>

Compression level for brotli: 0 (fastest, but worst compression) to 11 (best compression, but slowest)

--max-procs <max_procs>

Maximum number of simultaneous processes

--parallel, --serial

Whether to process input files in parallel or in series

--force, --no-force

Force all tasks to run, even those whose output files already exist

Arguments

INPUT_PATH

Optional argument(s)
